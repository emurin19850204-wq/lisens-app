# LISENS システム セキュリティ・コスト報告書

**報告日**: 2026年3月25日  
**プロジェクト名**: LISENS（社内ライセンス制度運用システム）  
**報告者**: 江村  
**ステータス**: 本番運用開始

---

## 1. システム概要

LISENSは、社内ライセンス制度（研修進捗管理・OSCE評価・認定管理）を一元管理するWebアプリケーションです。PC・スマートフォンのブラウザからアクセス可能で、専用アプリのインストールは不要です。

**本番URL**: https://element-lisense.netlify.app

---

## 2. セキュリティ対策

### 2.1 認証・アクセス制御

| 対策項目 | 実装状況 | 詳細 |
|---|---|---|
| パスワード認証 | ✅ 実装済み | メールアドレス＋パスワードによるログイン必須 |
| セッション管理 | ✅ 実装済み | Supabase Auth による自動セッション管理・トークン更新 |
| ロール別アクセス制御 | ✅ 実装済み | 5段階のロール（本部管理者・教育責任者・評価者・店舗責任者・研修者） |
| 行レベルセキュリティ（RLS） | ✅ 実装済み | データベースレベルでのアクセス制限 |

### 2.2 ロール別データアクセス範囲

| ロール | 閲覧範囲 | 編集権限 |
|---|---|---|
| 本部管理者 | 全データ | 全操作可能 |
| 教育責任者 | 全データ | 全操作可能 |
| 評価者 | 全研修者データ | 評価作成のみ |
| 店舗責任者 | 全研修者データ | 評価作成のみ |
| 研修者 | 自分のデータのみ | 閲覧のみ |

### 2.3 通信・インフラセキュリティ

| 対策項目 | 実装状況 | 詳細 |
|---|---|---|
| HTTPS暗号化通信 | ✅ 対応済み | Netlify による SSL/TLS 証明書自動管理 |
| 未ログインユーザーのブロック | ✅ 対応済み | API レベルでデータアクセスを完全遮断 |
| データベース暗号化 | ✅ 対応済み | Supabase（PostgreSQL）のストレージ暗号化 |
| DDoS 対策 | ✅ 対応済み | Netlify CDN による基本的なDDoS緩和 |

### 2.4 セキュリティ上の留意事項

- 現時点ではカスタムドメイン（例: lisens.element.co.jp）は未設定ですが、Netlify提供ドメインでもSSL通信は有効です
- 多要素認証（MFA）は未実装ですが、社内利用の現段階では必須ではありません。将来的に追加可能です
- パスワードポリシー（文字数・複雑さ等）はSupabase標準設定（最低6文字）に準じています

### 3.2 無料プランの利用制限

| サービス | 制限項目 | 無料枠 | 現在の使用量 |
|---|---|---|---|
| Supabase | データベース容量 | 500 MB | 約 1 MB（余裕あり） |

| Supabase | 認証ユーザー数 | 50,000 | 数名 |

### 3.3 スケールアップ時の費用見込み

| 条件 | 想定コスト | 備考 |
|---|---|---|
| 利用者 200名以下 | **¥0/月** | 無料枠内で対応可能（同時接続200まで、登録ユーザー50,000まで） |
| 利用者 200名超 or DB容量増大 | **約 ¥3,800/月** | Supabase Pro プランへの移行が目安 |
| カスタムドメイン追加 | **¥0〜¥2,000/月** | ドメイン取得費用のみ |

### 3.4 注意事項

- Supabase無料プランでは、**7日間アクセスがない場合にデータベースが一時停止**されます。日常的に利用していれば発生しません。停止した場合も、管理画面から即座に再開可能です
- 将来的にユーザー数や機能が拡大した場合でも、Supabase Pro（$25/月）への移行で十分対応できます

---

## 4. まとめ

LISENSは、**初期費用・ランニングコストともにゼロ**で本番運用を開始できる構成となっています。セキュリティ面では、パスワード認証・暗号化通信・ロール別アクセス制御を実装しており、**社外からのアクセスにも対応したセキュリティ水準**を確保しています。

今後の利用拡大に合わせて、段階的にプランをアップグレードすることで、コストを最小限に抑えながら運用を継続できます。
